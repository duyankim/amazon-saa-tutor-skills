# 하이브리드 네트워킹 (★★)

#aws-saa #networking

## 연결 옵션 비교

| 방법 | 대역폭 | 안정성 | 지연 | 보안 | 설치 시간 |
|------|--------|--------|------|------|---------|
| **Site-to-Site VPN** | 최대 1.25Gbps | 인터넷 의존 | 가변 | IPSec 암호화 | 수분 |
| **Direct Connect** | 1~100Gbps | 전용 회선 | 일관됨 | 비암호화* | 수주~수개월 |
| **Direct Connect + VPN** | 높음 | 높음 | 일관 | IPSec | 수주 |

*Direct Connect 자체는 암호화 없음. 암호화 필요 시 VPN over DX

```
온프레미스
   ├─ VPN (인터넷) ─────────────────→ VGW → VPC
   ├─ Direct Connect (전용 회선) ─→ DXG → VPC
   └─ Direct Connect + VPN ────────→ 보안+안정성
```

---

## Transit Gateway (TGW)

```
VPC-A ─┐
VPC-B ─┤
VPC-C ─┼─ Transit Gateway ─ VPN ─ 온프레미스
VPC-D ─┤
VPC-E ─┘
```

- N개 VPC를 허브로 연결
- **전이적 라우팅** 지원 (VPC Peering 불가)
- 멀티캐스트, AWS RAM으로 공유

---

## Direct Connect Gateway

- 하나의 DX 연결로 **여러 리전 VPC** 접근
- BGP 라우팅 사용

---

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "온프레미스 ↔ AWS 빠른 연결" | **Site-to-Site VPN** |
| "전용 회선, 안정적 대역폭" | **Direct Connect** |
| "Direct Connect + 암호화" | **VPN over Direct Connect** |
| "여러 VPC + 온프레미스 허브" | **Transit Gateway** |
| "DX로 여러 리전 VPC 접근" | **Direct Connect Gateway** |
