# 스토리지 마이그레이션 및 전략 (★★)

#aws-saa #migration

## 7R 마이그레이션 전략

| 전략 | 설명 | 노력 | 혜택 |
|------|------|------|------|
| **Retire** | 폐기 | 없음 | 비용 절감 |
| **Retain** | 유지 (온프레미스) | 없음 | 위험 없음 |
| **Relocate** | VMware → VMware Cloud on AWS | 낮음 | 빠름 |
| **Rehost** | Lift & Shift (그대로 이전) | 낮음 | 빠름 |
| **Replatform** | 일부 최적화 (RDS 전환 등) | 중간 | 중간 |
| **Repurchase** | SaaS 전환 | 중간 | 중간 |
| **Refactor** | 재아키텍처 (서버리스 등) | 높음 | 최고 |

---

## Application Migration Service (MGN)

```
온프레미스 서버
     ↓ (MGN 에이전트)
[AWS MGN] (지속 복제)
     ↓
EC2 인스턴스 (테스트/컷오버)
```

- Lift & Shift 자동화
- 에이전트 기반 블록 레벨 복제
- 최소 다운타임 마이그레이션

---

## 대용량 데이터 전송 의사결정

```
데이터 크기 / 네트워크 대역폭 = 전송 시간

< 1주일 (충분한 인터넷): DataSync / S3 Transfer Acceleration
1주일 ~ 수개월 (느린 인터넷): Snowball Edge
> 수 PB: Snowmobile
```

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "최소 변경, 빠른 이전" | **Rehost (Lift & Shift)** |
| "EC2 → RDS, 일부 최적화" | **Replatform** |
| "온프레미스 → Lambda 재설계" | **Refactor** |
| "가장 빠른 마이그레이션 방법" | **Rehost + MGN** |
| "10PB 데이터 빠른 전송" | **Snowmobile** |
