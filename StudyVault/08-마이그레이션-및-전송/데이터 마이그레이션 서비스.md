## AWS DMS (Database Migration Service)

```
소스 DB ─── DMS 복제 인스턴스 ─── 대상 DB
(Oracle/MySQL/MSSQL 등)           (RDS/Aurora/Redshift 등)

이기종 마이그레이션: DMS + SCT (Schema Conversion Tool)
동기종 마이그레이션: DMS만으로 가능
```

- **CDC (Change Data Capture)**: 온라인 마이그레이션 (서비스 중단 없음)
- **Full Load + CDC**: 초기 전체 복사 후 변경사항 동기화

---

## DataSync

```
온프레미스 파일 서버 ─── DataSync Agent ─── AWS
(NFS/SMB)                                   S3 / EFS / FSx
```

- **에이전트** 필요 (온프레미스에 설치)
- 자동 암호화, 검증, 스케줄링
- Storage Gateway와 달리 **일회성/주기적 동기화**

---

## Snowball 제품군

| 제품 | 용량 | 사용 케이스 |
|------|------|------------|
| **Snowcone** | 8~14TB | 엣지 컴퓨팅, 소용량 |
| **Snowball Edge Storage** | 80TB | 대용량 오프라인 마이그레이션 |
| **Snowball Edge Compute** | 42TB + GPU | 엣지 컴퓨팅 |
| **Snowmobile** | 100PB | 데이터센터 규모 마이그레이션 |

> [!tip] Snowball vs DataSync
> 인터넷 속도 느림 + 대용량 → **Snowball**
> 인터넷 속도 충분 + 파일 동기화 → **DataSync**

## Storage Gateway

| 유형 | 인터페이스 | 스토리지 |
|------|----------|---------|
| **File Gateway** | NFS/SMB | S3 |
| **Volume Gateway** (Cached) | iSCSI | S3 (캐시: 온프레미스) |
| **Volume Gateway** (Stored) | iSCSI | 온프레미스 (백업: S3) |
| **Tape Gateway** | VTL | S3 Glacier |

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "Oracle → Aurora 온라인 마이그레이션" | **DMS + SCT** |
| "온프레미스 NFS → EFS 동기화" | **DataSync** |
| "인터넷 없이 수TB 데이터 전송" | **Snowball Edge** |
| "온프레미스 NAS → S3 백업" | **Storage Gateway File** |
| "테이프 백업 클라우드 전환" | **Storage Gateway Tape** |
| "SFTP → S3 자동 업로드" | **AWS Transfer Family** |
