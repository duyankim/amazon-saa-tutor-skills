# 비용 최적화 전략 (★★)

#aws-saa #cost-optimization

## Overview Table

| 전략 | 절감 방법 | 대상 |
|------|----------|------|
| Spot Instance | 최대 90% | 중단 허용 배치/분석 |
| Reserved Instance | 최대 72% (1년) | 안정적 EC2/RDS |
| Savings Plans | 최대 72% | 유연한 Compute/EC2/SageMaker |
| S3 계층화 | 접근 패턴 최적화 | S3 스토리지 |
| Right-sizing | 미사용 용량 제거 | EC2/RDS |
| 수명주기 정책 | 자동 계층 전환/삭제 | S3/EBS 스냅샷 |

---

## EC2 구매 옵션 비교

```
비용: On-Demand > Reserved = Savings Plans > Spot

On-Demand:  기준 비용, 약정 없음
Reserved:   1년 40% / 3년 72% 절감, 특정 인스턴스 타입
Savings:    1년/3년 시간당 사용량 약정, 더 유연
Spot:       최대 90% 절감, 중단 가능
```

---

## Savings Plans 유형

| 유형 | 유연성 | 적용 범위 |
|------|--------|---------|
| **Compute Savings Plans** | 최고 (인스턴스 타입/리전/OS/테넌시) | EC2, Fargate, Lambda |
| **EC2 Instance Savings Plans** | 중간 (리전+패밀리 고정) | EC2만 |
| **SageMaker Savings Plans** | 낮음 | SageMaker만 |

## 비용 모니터링 도구

| 도구 | 역할 |
|------|------|
| **Cost Explorer** | 비용 분석 + 7일 예측 |
| **AWS Budgets** | 임계값 초과 시 알림 |
| **Cost & Usage Report** | 상세 청구 보고서 |
| **Trusted Advisor** | 비용 최적화 권고 |
| **Compute Optimizer** | EC2/Lambda 인스턴스 최적화 권고 |

## S3 비용 최적화

```
접근 빈도 분석 → 수명주기 정책 설정:
Standard → Standard-IA (30일) → Glacier (90일) → Deep Archive (180일) → 삭제
또는
Intelligent-Tiering (자동 계층화)
```

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "비용 최소화 + 중단 허용 배치" | **Spot Instance** |
| "1년 약정, 가장 저렴" | **Reserved Instance (표준)** |
| "유연한 약정, EC2+Fargate+Lambda" | **Compute Savings Plans** |
| "비용 임계값 알림" | **AWS Budgets** |
| "EC2 인스턴스 크기 최적화 권고" | **Compute Optimizer** |
| "S3 자동 계층화" | **S3 Intelligent-Tiering** |
