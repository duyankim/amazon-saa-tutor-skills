# 계정 관리 (★★)

#aws-saa #security

## AWS Organizations

```
Root
├── OU (개발)
│   ├── 계정 A
│   └── 계정 B
└── OU (운영)
    ├── 계정 C
    └── 계정 D
```

- **SCP (Service Control Policy)**: OU/계정 레벨 최대 권한 설정
- **통합 결제**: 모든 계정 비용 통합 관리
- **볼륨 할인**: 전체 사용량 기반 할인

---

## SCP 핵심 규칙

> [!important] SCP는 최대 권한 경계
> - SCP는 Allow 정책이 아닌 **최대 허용 경계**
> - SCP로 허용된 것 + IAM Policy 허용된 것 = 실제 가능한 권한
> - SCP는 **root 계정**에도 적용됨 (관리 계정 제외)

---

## AWS Control Tower

- 멀티 계정 환경 **자동 설정 및 거버넌스**
- Guardrails: 예방(Preventive SCP) / 탐지(Detective Config 규칙)
- Landing Zone: 모범 사례 기반 초기 환경 구성

---

## AWS RAM (Resource Access Manager)

- VPC 서브넷, Transit Gateway, Route 53 Resolver 등 **리소스 공유**
- Organizations 계정 간 무료 공유

---

## Trusted Advisor

| 카테고리 | 확인 내용 |
|---------|---------|
| 비용 최적화 | 미사용 리소스, Reserved 구매 권고 |
| 성능 | 고활용 EC2, CloudFront 최적화 |
| 보안 | 퍼블릭 S3, MFA 미설정 root |
| 내결함성 | EBS 백업, RDS Multi-AZ |
| 서비스 한도 | 리소스 한도 근접 경고 |

> [!tip] Trusted Advisor 전체 기능
> Business/Enterprise 지원 플랜에서만 전체 Trusted Advisor 체크 사용 가능
