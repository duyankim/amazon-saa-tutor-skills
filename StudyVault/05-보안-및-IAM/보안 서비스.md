# 보안 서비스 (★★★)

#aws-saa #security

## Overview Table

| 서비스 | 목적 | 자동화 |
|--------|------|--------|
| **WAF** | L7 웹 방화벽 (SQL injection, XSS) | 규칙 기반 |
| **Shield Standard** | DDoS 기본 보호 | 자동, 무료 |
| **Shield Advanced** | 고급 DDoS + 비용 보호 | 자동, 24/7 DRT |
| **GuardDuty** | 위협 감지 (이상 행동) | ML 기반 |
| **Macie** | S3 민감 데이터 감지 | ML 기반 |
| **Inspector** | 취약점 스캔 | 자동 |
| **Security Hub** | 보안 경보 통합 대시보드 | 통합 |

---

## WAF 핵심

```
CloudFront / ALB / API Gateway / AppSync
     ↓
   [WAF]
     ↓
  백엔드 서버

WAF 규칙: IP 차단 / SQL injection / XSS / Rate Limiting
```

- **Web ACL**: 규칙 그룹 컨테이너
- **Managed Rules**: AWS/마켓플레이스 제공 사전 구성 규칙

---

## GuardDuty

분석 소스:
```
CloudTrail Logs     → API 호출 이상 감지
VPC Flow Logs       → 네트워크 이상 감지
DNS Logs            → 악성 도메인 쿼리
S3 Data Events      → S3 이상 접근
RDS Login Events    → DB 비정상 로그인
```

> [!tip] GuardDuty는 분석만
> 로그를 분석하여 위협 **감지**만 함. 차단은 WAF/Security Group 등 별도 조치 필요

---

## Inspector vs Macie

| 서비스 | 대상 | 탐지 내용 |
|--------|------|---------|
| **Inspector** | EC2, Lambda, ECR | CVE 취약점, 네트워크 노출 |
| **Macie** | S3 버킷 | PII(개인정보), 민감 데이터 |

---

## Cognito (사용자 인증)

| 구성요소 | 역할 |
|---------|------|
| **User Pool** | 사용자 인증 (로그인/회원가입) |
| **Identity Pool** | AWS 리소스 접근을 위한 임시 자격증명 |

```
사용자 → Cognito User Pool (로그인) → JWT 토큰
       → Cognito Identity Pool → IAM 임시 자격증명 → AWS 리소스
```

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "SQL injection 차단" | **WAF** |
| "DDoS 고급 보호 + 비용 보호" | **Shield Advanced** |
| "AWS 계정 이상 활동 감지" | **GuardDuty** |
| "S3 PII 데이터 감지" | **Macie** |
| "EC2 취약점 자동 스캔" | **Inspector** |
| "웹앱 사용자 로그인" | **Cognito User Pool** |
| "소셜 로그인 → AWS 리소스 접근" | **Cognito Identity Pool** |
