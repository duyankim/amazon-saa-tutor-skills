# S3 고급 기능 (★★★)

#aws-saa #storage

## Overview Table

| 기능 | 목적 |
|------|------|
| 교차 리전 복제(CRR) | 다른 리전 버킷으로 자동 복제 |
| 동일 리전 복제(SRR) | 같은 리전 버킷으로 복제 (로그 통합 등) |
| Transfer Acceleration | CloudFront 엣지 경유 업로드 가속 |
| Presigned URL | 임시 접근 URL 생성 |
| 이벤트 알림 | S3 → SNS/SQS/Lambda 트리거 |
| S3 Select | SQL로 객체 내 데이터 추출 |

---

## 복제 (Replication)

```
소스 버킷 (버전 관리 필수)
     ↓ CRR (Cross-Region)
대상 버킷 (버전 관리 필수)
```

> [!important] 복제 전제 조건
> - 소스/대상 버킷 모두 **버전 관리 활성화** 필수
> - 복제는 **신규 객체**에만 적용 (기존 객체 → S3 Batch Replication)
> - 삭제 마커는 기본 복제 안 됨 (옵션으로 활성화 가능)

---

## Transfer Acceleration vs DataSync vs Direct Connect

| 방법 | 경로 | 사용 케이스 |
|------|------|------------|
| **S3 Transfer Acceleration** | 인터넷 → CloudFront 엣지 → AWS 백본 → S3 | 글로벌 업로드 가속 |
| **DataSync** | 온프레미스 에이전트 → AWS | 온프레미스 파일 동기화 |
| **Direct Connect** | 전용 물리 회선 | 대용량/안정적 전송 |
| **Multipart Upload** | S3 직접 | 100MB+ 대용량 파일 |

---

## S3 이벤트 알림 패턴

```
S3 PUT/DELETE
    → SNS  (여러 구독자에게 알림)
    → SQS  (메시지 큐로 비동기 처리)
    → Lambda (직접 처리)
    → EventBridge (고급 필터링/라우팅)
```

---

## S3 Object Lock (WORM)

- **Governance Mode**: 특정 권한 없으면 덮어쓰기/삭제 불가
- **Compliance Mode**: root 포함 아무도 삭제 불가 (규정 준수)
- **Legal Hold**: 보존 기간 없이 Lock 유지

> [!tip] 규정 준수 요구시
> "변경/삭제 불가" + "규정 준수" → **S3 Object Lock Compliance Mode**

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "글로벌 사이트 → S3 빠른 업로드" | **S3 Transfer Acceleration** |
| "다른 리전에 자동 복제" | **S3 CRR** |
| "임시로 S3 파일 외부 공유" | **Presigned URL** |
| "S3 업로드 시 처리 자동화" | **S3 Event → Lambda/SQS** |
| "규정상 삭제 불가 데이터" | **S3 Object Lock Compliance** |
| "S3 내 특정 컬럼만 추출" | **S3 Select** |
