## 스토리지 클래스 비교

| 클래스 | 가용성 | AZ | 접근 빈도 | 최소 저장 | 검색 비용 |
|--------|--------|-----|----------|----------|----------|
| **Standard** | 99.99% | ≥3 | 자주 | 없음 | 없음 |
| **Standard-IA** | 99.9% | ≥3 | 비자주 | 30일 | 있음 |
| **One Zone-IA** | 99.5% | 1 | 비자주 | 30일 | 있음 |
| **Intelligent-Tiering** | 99.9% | ≥3 | 혼합 | 없음 | 없음* |
| **Glacier Instant** | 99.9% | ≥3 | 분기 | 90일 | 있음 |
| **Glacier Flexible** | 99.99% | ≥3 | 연간 | 90일 | 있음 |
| **Glacier Deep Archive** | 99.99% | ≥3 | 거의 없음 | 180일 | 있음 |

> [!warning] One Zone-IA 주의
> AZ 장애 시 데이터 손실 가능 → 재현 가능한 데이터에만 사용

> [!tip] Intelligent-Tiering
> 모니터링 비용 발생하지만 검색 비용 없음. 접근 패턴 불규칙하고 객체 수 많을수록 유리

## S3 보안 설정

```
퍼블릭 차단:  Block Public Access (계정/버킷 레벨)
버킷 정책:    JSON 기반 리소스 정책 (교차 계정 포함)
ACL:         레거시, 기본 비활성화 권장
암호화:       SSE-S3 / SSE-KMS / SSE-C / DSSE-KMS
```

| 암호화 유형 | 키 관리 | 특징 |
|------------|--------|------|
| **SSE-S3** | AWS | 기본값, 자동 |
| **SSE-KMS** | KMS CMK | 감사 추적, 세밀한 제어 |
| **SSE-C** | 고객 | 고객이 키 제공 |
| **DSSE-KMS** | KMS | 이중 암호화 |

## S3 버전 관리 & MFA Delete

- **버전 관리**: 삭제 마커(Delete Marker) 생성 → 이전 버전 복구 가능
- **MFA Delete**: 버전 영구 삭제 시 MFA 인증 필요

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "자주 접근, 최고 가용성" | **S3 Standard** |
| "비자주 접근, 다중 AZ" | **S3 Standard-IA** |
| "비자주 접근, 저렴, 재현 가능" | **S3 One Zone-IA** |
| "접근 패턴 모름" | **S3 Intelligent-Tiering** |
| "분기별 밀리초 검색" | **S3 Glacier Instant** |
| "연간 몇 번, 12시간 내 검색" | **S3 Glacier Deep Archive** |
| "교차 계정 S3 접근" | **버킷 정책 (Resource Policy)** |
