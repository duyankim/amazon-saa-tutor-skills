# 고가용성과 재해복구 전략 (★★★)

#aws-saa #ha-dr

## RTO vs RPO

```
재해 발생          복구 완료
    ↓                  ↓
────●──────────────────●──────→ 시간
    ↑                  ↑
  RPO (←───────────────)    = 허용 가능한 데이터 손실 시간
                   RTO = 시스템 복구 소요 시간
```

| 지표 | 정의 | 낮을수록 |
|------|------|---------|
| **RPO** (Recovery Point Objective) | 허용 가능한 데이터 손실 시간 | 더 자주 백업 |
| **RTO** (Recovery Time Objective) | 허용 가능한 복구 시간 | 더 많은 리소스 대기 |

---

## DR 전략 비교

| 전략 | RTO | RPO | 비용 | 설명 |
|------|-----|-----|------|------|
| **Backup & Restore** | 수시간 | 수시간 | 최저 | S3 백업 후 복원 |
| **Pilot Light** | 수십분 | 분 | 낮음 | 핵심 시스템만 최소 실행 |
| **Warm Standby** | 분 | 초~분 | 중간 | 소규모 프로덕션 환경 상시 가동 |
| **Active-Active** | 실시간~0 | ~0 | 최고 | 멀티 리전 동시 운영 |

```
비용 ↑                          RTO/RPO ↓
Backup ──→ Pilot Light ──→ Warm Standby ──→ Active-Active
```

## 서비스별 HA 구성

| 서비스 | HA 방법 |
|--------|--------|
| EC2 | Auto Scaling + Multi-AZ ALB |
| RDS | Multi-AZ (동기 복제, 자동 장애조치) |
| Aurora | 3 AZ 6 사본 자동 복제 |
| S3 | 기본 다중 AZ 내구성 11 9's |
| DynamoDB | 기본 다중 AZ, Global Tables (멀티 리전) |
| EFS | 다중 AZ 스토리지 |

## 멀티 리전 아키텍처

```
리전 A (Primary)          리전 B (Secondary)
  EC2 + RDS              EC2 + RDS Read Replica
     ↕ CRR                    ↕ (장애 시 승격)
  S3 버킷  ──────────────→  S3 버킷 (CRR)
     ↕                        ↕
Route 53 Failover ──────────────
```

## AWS Backup

- 중앙화된 백업 관리
- 지원: EC2, EBS, EFS, RDS, Aurora, DynamoDB, FSx, Storage Gateway
- **교차 계정/교차 리전 백업** 가능
- 보존 정책, 백업 계획 자동화

## Exam/Test Patterns

| 시나리오/키워드 | 답 |
|----------------|-----|
| "최저 비용 DR" | **Backup & Restore** |
| "핵심만 켜놓고 빠른 복구" | **Pilot Light** |
| "RTO 몇 분, 비용 중간" | **Warm Standby** |
| "RPO ≈ 0, 무중단" | **Active-Active Multi-Region** |
| "RDS 자동 장애복구" | **RDS Multi-AZ** |
| "멀티 리전 DynamoDB" | **DynamoDB Global Tables** |
