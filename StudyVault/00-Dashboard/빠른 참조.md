## 컴퓨팅 → [[EC2 기초]] [[Lambda와 서버리스]] [[컨테이너 ECS-EKS]] [[Auto Scaling]]

| 서비스/개념 | 핵심 포인트 |
|------------|------------|
| EC2 Spot | 최대 90% 절감, 인터럽트 가능 — 배치 작업/비중단 워크로드 |
| EC2 Reserved | 1~3년 약정, 최대 72% 절감 — 안정적 워크로드 |
| EC2 On-Demand | 약정 없음, 유연 — 단기/예측 불가 워크로드 |
| Savings Plans | Compute/EC2/SageMaker — 시간당 사용량 약정 |
| Auto Scaling | 수평 확장 (인스턴스 추가), ELB와 연동 |
| Lambda | 최대 15분, 최대 10GB 메모리, 동시성 1000 (기본) |
| ECS Fargate | 서버리스 컨테이너 — 인프라 관리 불필요 |
| EKS | 관리형 Kubernetes — 복잡한 오케스트레이션 |
| Elastic Beanstalk | PaaS — 인프라 자동 관리, 코드만 업로드 |

## 스토리지 → [[S3 핵심 개념]] [[S3 고급 기능]] [[EBS-EFS-FSx]] [[Glacier와 수명주기]]

| 서비스/개념 | 핵심 포인트 |
|------------|------------|
| S3 Standard | 자주 접근, 가용성 99.99%, 내구성 11 9's |
| S3 Standard-IA | 비자주 접근, 검색 비용 발생 |
| S3 One Zone-IA | 단일 AZ, 20% 저렴, 재현 가능 데이터 |
| S3 Intelligent-Tiering | 접근 패턴 자동 감지 — 모니터링 비용 발생 |
| S3 Glacier Instant | 밀리초 검색, 분기별 접근 |
| S3 Glacier Flexible | 1분~12시간 검색 (Expedited/Standard/Bulk) |
| S3 Glacier Deep Archive | 12~48시간, 최저가 — 7~10년 보관 |
| S3 Transfer Acceleration | CloudFront 엣지 경유 업로드 속도 향상 |
| EBS gp3 | 범용 SSD, 최대 16,000 IOPS, EC2에 1:1 연결 |
| EBS io2 | 고성능 SSD, 최대 256,000 IOPS, 멀티-어태치 |
| EFS | 관리형 NFS, 다수 EC2 동시 마운트, Linux only |
| FSx for Windows | SMB 프로토콜, AD 통합 |
| FSx for Lustre | HPC/ML 워크로드, S3 통합 |

## 데이터베이스 → [[RDS와 Aurora]] [[DynamoDB]] [[ElastiCache]] [[기타 데이터베이스]]

| 서비스/개념 | 핵심 포인트 |
|------------|------------|
| RDS Multi-AZ | 동기 복제, 자동 장애조치 — 고가용성 |

| Aurora Serverless | 간헐적 워크로드, 자동 스케일링 |
| DynamoDB | 키-값/문서, 서버리스, 단일 ms 지연 |

| DynamoDB DAX | 마이크로초 지연 캐시 (DynamoDB 전용) |
| ElastiCache Redis | 다중 AZ, Pub/Sub, 정렬셋, 지속성 |
| ElastiCache Memcached | 멀티스레드, 단순 캐시 — 지속성 없음 |
| Redshift | 열 기반 OLAP, 페타바이트급 분석 |

## 네트워킹 → [[VPC 핵심]] [[로드밸런서와 Route 53]] [[CloudFront와 API Gateway]] [[하이브리드 네트워킹]]

| 서비스/개념 | 핵심 포인트 |

| Route 53 Simple | 기본 라우팅, 헬스체크 없음 |
| Route 53 Weighted | 비율 기반 (A/B 테스트) |
| Route 53 Latency | 지연시간 최소 리전으로 라우팅 |
| Route 53 Failover | 기본/보조 전환, 헬스체크 필수 |
| Route 53 Geolocation | 사용자 위치 기반 |
| Route 53 Multi-Value | 최대 8개 헬시 레코드 반환 |
| CloudFront | CDN, 엣지 캐싱, S3/ALB/EC2 오리진 |
| VPC Peering | 1:1 VPC 연결, 전이적 피어링 불가 |
| Transit Gateway | N:N VPC 허브, 전이적 라우팅 지원 |
| Direct Connect | 전용 물리 회선, 안정적 대역폭 |
| Site-to-Site VPN | 인터넷 기반 암호화 터널 |
| Global Accelerator | Anycast IP, AWS 백본 경유, 고정 IP |

## 애플리케이션통합 → [[SQS와 SNS]] [[Kinesis]] [[EventBridge와 Step Functions]]

| 서비스/개념 | 핵심 포인트 |
|------------|------------|
| SQS Standard | At-least-once, 순서 보장 없음, 높은 처리량 |
| SQS FIFO | Exactly-once, 순서 보장, 최대 300TPS |
| SQS DLQ | 처리 실패 메시지 격리 — maxReceiveCount |
| SNS | Pub/Sub, 팬아웃 패턴, 다수 구독자 동시 알림 |
| SNS+SQS 팬아웃 | 1 SNS → N SQS → N 컨슈머 (권장 패턴) |
| Kinesis Data Streams | 실시간 스트리밍, Shard 단위, 1MB/s per shard |
| Kinesis Data Firehose | 완전관리형 ETL → S3/Redshift/OpenSearch |
| Kinesis Data Analytics | SQL로 실시간 스트림 분석 |
| EventBridge | 이벤트 버스, 스케줄링, SaaS 통합 |
| Step Functions | 서버리스 워크플로우 오케스트레이션 |
